# DESIGN-SYSTEM.md — b2b-dashboard UI language

> **Status:** v1, 2026-06-26. Authored before the fullscreen overhaul (see "Change map" at the
> bottom). This doc is the source of truth for *how this UI should look and feel*. It is also written
> to be reusable as a **system-prompt brief** for any future AI-assisted UI work on this repo — paste
> the "Hard rules" section into the model so it builds from our intent, not from training-data averages.

---

## 0. Why this doc exists

The brief: make the dashboard a **fullscreen, elegant, simple, genuinely-designed** product surface —
explicitly *not* something that reads as "generated by an AI." Two reference screenshots drive the
target look (the MoodScale admin dashboard + the Corporate dashboard): full-width canvas, a narrow
**dark-navy icon rail** on the left, a slim dark topbar, light cards floating on a soft slate canvas.

Two hard owner constraints sit on top:

1. **Avoid purple, yellow, and orange extensively.** No purple anywhere. Yellow/orange only ever as a
   *locked data-severity signal* (amber), never as chrome, decoration, or a badge that isn't data.
2. **No left/top accent bar and no border on any box.** Boxes are defined by **elevation (shadow)**,
   not outlines. No `border`, no `border-l-4`, no `border-t-*` accent strips on cards/panels/tiles.

## 1. What makes UI look "AI-generated" (research) — and how we beat it

Synthesised from a Reddit-mined dataset (3.2M posts / 47 subreddits) ranking the visual "tells" of
vibe-coded sites, plus design-systems-for-AI guidance. The recurring tells, most-cited first:

| # | The "tell" | Our counter-decision |
|---|------------|----------------------|
| 1 | **shadcn/Tailwind defaults left untouched** | We ship an opinionated token + component layer; no raw default card/button look. |
| 2 | **"AI purple" / purple→blue gradients** | Zero purple. Navy `#1E3A5F` + slate, one green accent. |
| 3 | **Gradient hero text** | Solid text only. Big numbers get weight + tight tracking + tabular-nums, never a gradient. |
| 4 | **Unprompted neon glow** | No glow. Depth comes from soft, layered, *low-opacity* shadows. |
| 5 | **Emoji used as icons** | Line icons, never emoji. Nav/chrome uses **Phosphor** (`@phosphor-icons/react`) — a deliberately non-default pack (lucide is the shadcn/AI default); content cards still use lucide. |
| 6 | **Centered hero + three cards** | Dashboard, not a landing page: intentional card-size hierarchy (hero KPI row + a wide primary chart), not a flat uniform grid. |
| — | Heavy glassmorphism (`backdrop-blur`) | Removed. It was a mild tell here; flat white surfaces read cleaner. |
| — | Aurora/mesh/blob backgrounds | None. A single restrained diagonal slate tint on the canvas only. |

**The deeper principle from the research:** *defaults look generic because they are averages.* The fix
is to encode **intent** as a constrained system (primitives → semantic → component tokens), give
positive references (not just "avoid X"), and make a few decisions only a human would make — deliberate
asymmetry, a real type scale, honest operational copy, uneven-but-rhythmic whitespace.

Sources:
- vibecoded-design-tells (Reddit-mined): https://github.com/JCarterJohnson/vibecoded-design-tells
- Design Systems for AI Coding / "Stop Getting Purple Gradients": https://www.braingrid.ai/blog/design-system-optimized-for-ai-coding
- Claude Design — Avoid Generic AI Aesthetics: https://www.mindstudio.ai/blog/claude-design-avoid-generic-ai-aesthetics
- Get Better UI from Claude (Research First): https://hassantayyab.com/blogs/claude-better-ui-research-first
- How to Actually Design with AI in 2026 (without generic garbage): https://www.giangallegos.com/how-to-actually-design-with-ai-in-2026-without-making-generic-garbage/

## 2. Layout — fullscreen + dark icon rail

```
┌────┬───────────────────────────────────────────────┐
│ ▓▓ │  topbar (dark navy): logo · title · workspace  │  ← slim, full width
│ ▓▓ │         switcher · MFA chip · avatar/sign-out  │
│ ▓▓ ├───────────────────────────────────────────────┤
│ ▓▓ │  content fills the viewport (soft slate canvas)│
│ ▓▓ │  ┌ KPI ┐┌ KPI ┐┌ KPI ┐┌ KPI ┐                 │
│ ▓▓ │  ┌────── wide chart ──────┐┌─ feed ─┐          │
│ ▓▓ │  └ tables / cards … ──────┘└────────┘          │
└────┴───────────────────────────────────────────────┘
  ^ dark-navy icon rail (~64px), lucide icons, active = subtle tint, label tooltip on hover
```

- **No centered `max-w-7xl` column.** Content is edge-to-edge with comfortable gutters (`px-6/px-8`),
  capped only very wide (e.g. `2xl:max-w-[1600px]`) so ultra-wide monitors don't stretch text lines.
- **Left rail** (`~64px`, `bg-[#1E3A5F]`): icon-only, vertically stacked, scrollable. Active item =
  lighter navy-tint pill behind the icon; hover = same at lower opacity; each shows its label as a
  styled hover/focus tooltip. Two groups split by a divider: **(top)** the seven dashboard tabs
  (Overview · Health & Risk · Engagement · Impact · Verify · Act & Programmes · Reports & Govern),
  each → `/dashboard?tab=<id>` with active state from the `tab` query param; **(bottom)** the other
  surfaces (Health Report, Participation, Onboarding, How it works, Evidence). The in-body horizontal
  `TabNav` is therefore desktop-removed — it renders only on phones (`md:hidden`), where the rail is
  hidden and a bottom bar (Dashboard + surfaces) takes over.
- **Topbar** (dark navy, `position: fixed` top-0, offset left by the rail width on md+ — NOT
  `sticky`, which drifted/left a growing gap on some pages; `<main>` carries matching `pt-20` so
  content never hides under it): brand logo (white-label aware),
  surface title, `WorkspaceSwitcher`, "MFA verified" chip, avatar / sign-out.
- **Canvas**: the existing soft diagonal slate tint stays (subtle, not a mesh/blob), so white cards
  read as raised paper.

### 2.1 Responsive behaviour (mobile + tablet are first-class)

The fullscreen rail layout must degrade cleanly down to phones. Breakpoints follow Tailwind
(`sm 640 · md 768 · lg 1024 · xl 1280 · 2xl 1536`).

| Surface | Phone (`< md`) | Tablet (`md–lg`) | Desktop (`≥ lg`) |
|---|---|---|---|
| **Left icon rail** | Hidden; nav moves to a **bottom tab bar** (fixed, dark navy, icon + tiny label, safe-area inset) | Visible icon rail (`~56px`) | Visible icon rail (`~64px`) |
| **Topbar** | Compact: logo + avatar + menu; title hidden, workspace switcher in an overflow/sheet | Logo + title + switcher + avatar | Full topbar |
| **Content gutters** | `px-4` | `px-6` | `px-8` |
| **KPI row** | 1 col → `grid-cols-2` from `sm` | `md:grid-cols-2` | `lg:grid-cols-4` |
| **2-up chart/table rows** | stacked (1 col) | usually 1 col | `lg:grid-cols-2` |
| **Tabs** (`TabNav`) | horizontal-scroll row (no wrap jank), momentum scroll | wrap/scroll | inline |
| **Tables** | horizontal scroll inside the card (`overflow-x-auto`), never squashed; key column sticky if needed | scroll as needed | full |
| **Bottom tab bar** | adds `pb-16` to `<main>` so content clears it | n/a | n/a |

Rules:
- **Dashboard filters** (Period/Department/Team/Level) render in the **app bar on `xl+`** as
  compact dark-tone dropdowns (`Dropdown tone="dark"`, no text labels); below `xl` they fall back to
  the body `FilterBar`. Both bind the same URL params via `useUrlFilters`, so they stay in sync.
- Tap targets ≥ 44px on touch; rail/bottom-bar icons get adequate padding.
- Respect iOS safe areas: bottom bar uses `pb-[env(safe-area-inset-bottom)]`.
- No fixed pixel widths that overflow small screens; cards are fluid, charts use a responsive container.
- Test at 375 (phone), 768 (tablet portrait), 1024 (tablet landscape), 1440 (desktop).

## 3. Elevation — borderless boxes (the core rule)

**Every box is defined by shadow, never by a border or accent strip.**

- `Card` and every panel/tile: **no `border`, no `backdrop-blur`.** White surface + a two-layer soft
  shadow, lifting slightly on hover.
- **No `border-l-4` / `border-t` accent strips.** State signalling that used a colored left bar moves
  to: a small **status dot**, a **soft-tinted background wash**, or a **badge** — never a bar.
- Inner sub-tiles (e.g. KPI metric tiles inside a card) that used `border border-brand-border` become
  **filled tiles** (`bg-brand-bg`), so they read as recessed wells, not outlined boxes.
- Hairline **table row dividers** (`border-t border-black/5`) are *table affordances, not boxes* — kept,
  but minimal. (If the owner wants them gone too, switch tables to zebra striping; flagged, not done.)

### Shadow tokens (added to `globals.css`)
```
--shadow-card:        0 1px 2px rgba(16,24,40,.04), 0 8px 24px -12px rgba(16,24,40,.10);
--shadow-card-hover:  0 1px 2px rgba(16,24,40,.05), 0 12px 28px -10px rgba(16,24,40,.14);
```
Exposed to Tailwind as `shadow-card` / `shadow-card-hover` via `@theme inline`.

## 4. Palette

Two **strictly separated** palettes (unchanged principle, doc 10 §2.1) — but chrome is now navy/slate
only, and the severity colors are quarantined to *data*.

**(a) Brand / chrome** — themeable per org:
- `--brand-primary` navy `#1E3A5F` — rail, topbar, active states, primary buttons, focus rings.
- slate neutrals: `--brand-bg #f8fafc`, `--brand-surface #fff`, `--brand-text #0f172a`,
  `--brand-muted #475569`, `--brand-border #e2e8f0` (border token now used only for hairlines/dividers).

**(b) Data-severity** — FIXED, never themed, never used as chrome/decoration:
- `--severity-green #22c55e` (positive / the single sanctioned accent that may appear as success chrome)
- `--severity-amber #f59e0b` (caution — **data only**)
- `--severity-coral #f08e80` (serious, calm ramp top — **data only**, never red)
- `--severity-red #e84d3d` (discrete employer threshold alert **only**)

**Owner constraint mapping:**
- **Purple** → banned outright. (None present today; keep it that way.)
- **Yellow/Orange** → only `--severity-amber`/`--severity-coral` as a *data band*. Any amber/orange in
  chrome (e.g. `text-amber-600` decoration) is removed.
- **Green** is the only severity tone allowed to double as chrome (success/MFA/celebration), because it
  reads as "good," not as an alert.

## 5. Typography & numbers

- Inter (already wired) — but escape the all-`text-sm` flatness. Use a real scale:
  page title `text-2xl font-bold tracking-tight`; card titles `text-sm font-medium text-brand-muted`;
  hero numbers `text-3xl/4xl font-semibold tabular-nums tracking-tight`.
- All figures use `tabular-nums` so columns align (already common — make it universal).
- Copy stays honest/operational (the "honest-or-pending" voice) — that specificity is itself an
  anti-generic signal.

---

## 6. Components (the opinionated layer)

- **Card** — borderless, `bg-brand-surface`, `rounded-xl`, `shadow-card`, `hover:shadow-card-hover`,
  `transition-shadow`. No blur. (`ui/primitives.tsx`)
- **Badge** — pill, soft solid fill via `color-mix` (kept), `text-xs font-semibold`. Used for state, not
  decoration.
- **Button** — navy primary / outline / ghost (kept). Outline uses a hairline, which is fine (it's a
  control, not a box).
- **StatusDot** *(new, small)* — replaces left-bar accents: a 6–8px filled dot in the severity tone +
  optional label. Used by `WidgetShell`, `GuardrailAlertStrip`, `IncidentsPanel`, the Overview banner.
- **Skeleton / CardError** — unchanged in behaviour; CardError stays visually distinct from suppression.

## 7. Hard rules (paste this block as a system-prompt brief)

```
DESIGN RULES — b2b-dashboard (MoodScale employer UI)
- Layout: fullscreen, edge-to-edge. Left dark-navy (#1E3A5F) icon rail (~64px) + slim dark topbar.
  No centered max-w column. Cap content only on ultra-wide (2xl:max-w-[1600px]).
- Boxes: NO borders, NO left/top accent bars, NO backdrop-blur. Depth = soft layered shadow
  (shadow-card / shadow-card-hover). Inner tiles = filled (bg-brand-bg), not outlined.
- Color: navy + slate chrome ONLY. The single accent is green (#22c55e) for success.
  BANNED: purple/violet/indigo (any). Amber/coral/red appear ONLY as data-severity signals
  (a real metric band or a discrete threshold alert) — never as chrome, decoration, or a non-data badge.
- Icons: line icons, never emoji. Nav/chrome = Phosphor (@phosphor-icons/react), duotone weight (active = fill) — a non-default pack so the rail doesn't read as AI-default lucide. Content cards keep lucide.
- Type: Inter, real scale (not all text-sm). Numbers use tabular-nums + tracking-tight; no gradient text.
- No glow, no glassmorphism, no aurora/mesh/blob backgrounds, no gradient hero text.
- State signalling uses a status dot / tinted wash / badge — never a colored bar.
- Tokens only: prefer brand-/severity- semantic classes; avoid arbitrary text-[#hex]/p-[13px].
- Privacy invariant unchanged: aggregate (k≥5) + participation status only; never an individual number.
```

## 8. Change map (file → what changes in this overhaul)

**Centralized (highest leverage):**
- `src/app/globals.css` — add `--shadow-card`/`--shadow-card-hover` (+ `@theme inline` exposure); keep
  severity tokens; canvas tint unchanged.
- `src/components/ui/primitives.tsx` — `Card`: drop `border border-brand-border` + `backdrop-blur-sm`,
  use `shadow-card`/`hover:shadow-card-hover`. Add `StatusDot`. (Fixes most boxes at once.)
- `src/components/layout/AppShell.tsx` — full rework to fullscreen + dark-navy icon rail + slim topbar.
- `src/components/layout/WorkspaceSwitcher.tsx` — borderless segmented control (drop the outline).

**Borderless / accent-strip sweep:**
- `src/components/dashboard/FilterBar.tsx`, `DataConfidenceStrip.tsx` — drop border + blur → `shadow-card`.
- `src/components/dashboard/WidgetState.tsx` — replace `border-l-4` alert/celebration accent with StatusDot.
- `src/components/dashboard/GuardrailAlertStrip.tsx` — drop `border-l-4`; signal via dot + badge; inner
  violation `<li>` loses its border → filled tile.
- `src/components/dashboard/IncidentsPanel.tsx`, `ProofEngineCard.tsx`, `RiskImpactCard.tsx`,
  `tabs/ActTab.tsx`, `tabs/VerifyTab.tsx`, `tabs/OverviewTab.tsx` (state banner) — remove `border`/
  `border-l-4`; use shadow + (where a tone is meaningful) a soft tinted wash / dot.
- `src/components/dashboard/OrdiCard.tsx` — remove chrome `text-amber-600` (keep amber only if it is the
  data band; otherwise neutral muted text).
- Inner bordered tiles (`KciScorecardCard.tsx` metric/lifecycle tiles, `MetricCellGrid`, `MotionQuadrant`,
  `InsightRail`, `ThermoclineCard`, `HelpSeekingCard`, etc.) — `border border-brand-border` → `bg-brand-bg`.
- Pages: `(app)/teams/page.tsx`, `(app)/guide/page.tsx`, `(app)/report/page.tsx`,
  `(app)/evidence/page.tsx` — same rules (report keeps its print hairlines; it's a document, not chrome).

**Untouched on purpose:** severity logic (`severity.ts`), data-band amber/coral usage, the privacy
footer/watermark, all GraphQL/hooks/data behaviour. This is a *visual* overhaul only.
